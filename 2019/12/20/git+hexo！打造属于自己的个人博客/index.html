git+hexo！打造属于自己的个人博客
2019-12-20

使用git+hexo的配置，我们可以很轻易的打造一个简洁美观、功能强大的个人博客！

不过首先，我们有必要了解下其中的原理：

  • git是什么

这里说的Git主要是指GitHub Pages，GitHub Pages本用于托管在GitHub的项目，由于他的空间免费稳定，
非常适合做博客。

  • hexo又是什么

Hexo是一个开源的静态博客生成器,用node.js开发,作者是台湾大学生tommy351,鉴于其静态架构，
用来做博客非常简洁，性能好。

  • git+hexo能起到什么作用

通过将Hexo生成hexo静态网站文件并上传到GitHub 实现免费的个人博客。

1、前期准备

  • 一个github账号

可自行到官网注册，这里不做过多赘述

  • git for window的下载

参考git下载与安装教程

  • node.js的下载及安装

在node官网中选择相应的版本

安装时直接默认下一步

通过在dos中输入：node -v检验安装是否成功

2、github生成自己的专属博客

  • 新建一个名为username.github.io的仓库

如本人的github用户名为MajyoSeika，那么就应该新建名为MajyoSeika.github.io的仓库，将来的网站访问地址即为http://majyoseika.github.io

注：这也意味着一个github账户最多只能创建一个这样可以使用域名访问的仓库

  • 配置SSH key

为什么要配置这个呢？因为你提交代码肯定要拥有你的github权限才可以，但是直接使用用户名和密码太不安全了，所以我们使用ssh key来解决本地和服务器的连接问题。

如果提示：No such file or directory 说明你是第一次使用git。

然后连续3次回车，最终会生成一个文件在用户目录下，打开用户目录，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key：

将刚复制的内容粘贴到key那里，title随便填，保存。

注：该ssh秘钥只出现一次，需妥善保存，下次再想找到并不轻松

  • 测试是否成功

如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到：

看到这个信息说明SSH已配置成功！

3、hexo的安装

不同于其他第三方软件，hexo的安装既可以通过window的cmd来完成，也可以使用git bash来完成，但是部分命令会有一些问题，为避免不必要的问题，建议全部使用git bash来执行。

  • 通过npm来进行hexo安装

  • hexo的初始化

在电脑的某个地方新建一个名为hexo的文件夹（名字可以随便取），比如我的是D:\hexo，由于这个文件夹将来就是作为你存放代码的地方，所以最好不要随便放不相关的文件。

hexo会自动下载一些文件到这个目录，包括node_modules，目录结构如下图：

themes文件夹存放你喜欢的主题，source/_posts文件夹存放你写好的md文件

4、hexo发布至github

写博客的最终目的当然是发布到网络与他人共同分享自己的成果，前面我们仅仅只能在本地预览，这里我们首先关联好自己的github仓库，然后再通过指令发布博客：

其中hexo generate是hexo g的缩写，hexo server是hexo s的缩写，hexo deploy是hexo d的缩写。

6、hexo主题的使用

可以看到，hexo提供给我们的默认主题还是比较简陋的，而hexo最大的特点便是可以使用许多大牛设计好的开源主题，然后通过修改打造自己的个性化网页！

  • 进入官方主页，在theme中可以看到许多主题
  • 遇到心仪的主题后，跳转至github仓库中将其拷贝到本地
  • 将下载好的压缩包解压，并放置在hexo/themes中
  • 最后修改_config.yml中的theme: landscape改为theme: 你的主题名字，然后重新执行hexo g→hexo d即可。

注：

  1. theme:后必须空一格，不然会出错

  2. 如果出现一些莫名其妙的问题，可以先执行hexo clean来清理一下public的内容，然后再来重新生成和发布。
