Git使用简述
2019-12-17

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。Git 是 Linus Torvalds为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。

毫无疑问，Git是程序员进步的一个有力武器，但是要想熟练地运用好它还是有一定的技巧了。其中最重要的便是关于它的代码托管了。

为此博主在这里记录下关于git的使用技巧，方便日后回顾：

一、Git下载

直接在百度搜索git官网，下载对应系统的版本即可：

二、Git安装

安装过程也很简单，不过有几个需要注意的点：

  1. 选择使用命令行模式
  1. 以下三步默认，直接点击下一步
  1. 安装完成，可通过鼠标右键检查
  1. 进行环境变量配置，并在dos界面中输入：git –version ，若出现如下结果，证明git安装成功

三、初始化操作

在使用git之前，必须设置与自己github账号相关联的账号及邮箱，右键选择Git Bash，进入命令行模式，以后的操作都在此进行：

  1. 设置用户名：git config –global user.name ‘name’

  2. 设置邮箱：git config –global user.email ‘email’

注：若其中有错误提示，可以尝试输入：git init（在当前目录生成一个隐藏的文件夹.git，该目录即是git仓库）

四、添加文件至仓库

在git中，为了方便管理文件，其添加文件的流程一般为：工作区→暂存区→Git仓库

可以通过：git status 查看git仓库的状态

  1. 将文件从工作区添加到暂存区：git add XXX(文件名)

此时通过查看仓库的状态，可以找到暂存区的文件

  1. 将在暂存区的文件添加至仓库：git commit -m ‘add XXX（对此次修改的描述）’，如

五、修改仓库文件

一般地，有两种方式修改工作区的文件：

  1. 直接打开文件进行修改

  2. 输入：vi XXX（文件名称）在命令行进行修改

注：此方法需掌握一些linux指令，这里不做过多赘述

六、删除仓库文件

同样有两种方式：

  1. 直接找到文件右键删除

  2. 在命令行模式中输入：rm XX（文件名称）

七、将本地仓库同步到远程仓库

在这一步，我们的最终目标是将本地代码上传至自己的github账号中进行托管

1、克隆github上的代码：

这里有两种操作：

  1. 直接在代码页面中点击download，拷贝到本地，如下图：

  1. 在命令行模式中输入：git clone 仓库地址（该地址可在代码页面中进行复制）

2、远程同步：

在git仓库下，输入：git push即可，成功与否可到自己的github中查看：

注意：

可能会出现同步失败（The requested URL returned error:403 Forbidden while accessing）

此时的解决方案为：

vi.git/config

将[remote ‘origin’]

url = https://github.com/用户名/仓库名.git

修改为

[remote ‘origin’]

url= https://用户名：密码@github.com/用户名/仓库名.git

八、一些常用的git指令

  1. 查看用户信息：git config –list

  1. 查看修改提交信息：git log

  1. 以图形化展示git的提交信息：git log –graph
  2. 查看指定版本信息：git show XXXXXXXXXX（show后面为每次提交系统后自动生成的哈希值）
  3. 回退到指定版本：git reset –hard XXXXXX（hard后面为每次提交系统后生成的哈希值）
  4. 回退到前n个修改：git reset HEAD^（回退1次）、git reset HEAD^^(回退2次)、git reset HEAD~3（回退3次），以此类推

更多git指令请参考网络
